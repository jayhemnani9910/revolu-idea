Quick start

Clone, install, configure .env, run:

git clone https://github.com/jayhemnani9910/revolu-idea.git
cd revolu-idea
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# copy template, then edit keys
cp .env.example .env

python main.py "What are the latest developments in quantum computing?"

No-key smoke run: LLM_PROVIDER=mock SEARCH_PROVIDER=mock

Key features

  • LangGraph multi-agent orchestration (planner → search → analyze → report → audit).
  • Multi-provider web search (Tavily, Exa, DuckDuckGo + Wikipedia fallback).
  • Adversarial-style verification loop to reduce hallucinations.
  • Knowledge graph building (entities + relationships) as research proceeds.
  • Ports/adapters design so providers can be swapped without touching the core flow.

Repo tour

  • main.py — CLI entrypoint (runs the research graph).
  • container.py — provider wiring (LLM/Search/Storage).
  • graph/ — LangGraph workflow construction.
  • agents/ — graph nodes + shared state.
  • ports/ + adapters/ — interfaces + implementations.
  • domain/ — core models (citations, evidence, reports, graphs).

Configuration

Settings load from .env (gitignored). Start with:

cp .env.example .env

Core env vars:

  • LLM_PROVIDER, LLM_BASE_URL, LLM_MODEL, LLM_API_KEY
  • SEARCH_PROVIDER (+ TAVILY_API_KEY or EXA_API_KEY if used)
  • MAX_RECURSION_DEPTH, MAX_INVESTIGATIONS_PER_EDGE
  • OUTPUT_DIR

Outputs

Artifacts are written under output/:

  • output/reports/*.md + *.json — final report
  • output/graphs/*.mmd + *.json — causal graph + Mermaid
  • output/checkpoints/*.json — saved state for runs

The report includes citations, findings, and verification status.

Providers

  • LLM: OpenAI-compatible APIs (Groq/DeepSeek/etc) via LLM_BASE_URL.
  • LLM (no key): LLM_PROVIDER=mock for local smoke runs.
  • Search: Tavily / Exa (keys) or SEARCH_PROVIDER=duckduckgo (free).
  • Storage: local filesystem (OUTPUT_DIR).

Local Ollama adapter exists but is currently disabled by default in container.py.

How it works

User query
  → research planning
  → web search (Tavily / Exa / DuckDuckGo)
  → evidence extraction + synthesis
  → knowledge graph building
  → report generation (Markdown + JSON)
  → audit/verification loop
  → final report + saved artifacts

The graph runs asynchronously and streams progress; outputs are saved via the storage adapter.

Architecture (Mermaid)

High-level wiring: the CLI calls a LangGraph workflow, which uses ports/adapters for LLM, search, and storage.

flowchart LR U[User query] --> CLI[main.py (CLI)] CLI --> C[container.py (wiring)] C --> G[graph/ (LangGraph workflow)] C -->|LLMPort| LLM[adapters/* LLM\nOpenAI-compatible or Mock] C -->|SearchPort| S[adapters/* Search\nTavily / Exa / DuckDuckGo] C -->|StoragePort| ST[adapters/local_storage.py\noutput/ artifacts] G --> N1[Plan] N1 --> N2[Search] N2 --> N3[Analyze] N3 --> N4[Build knowledge graph] N4 --> N5[Generate report] N5 --> N6[Audit/verify] N6 --> ST
